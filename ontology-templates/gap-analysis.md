## 갭 분석 방법

### Step 1: 클러스터 식별
full-wiki-ontology.md를 읽고 관계 밀도가 높은 노드 그룹을 클러스터로 식별

### Step 2: 클러스터 간 연결 강도 측정
| 강도 | 기준 |
|------|------|
| ■■■ (강) | 3+ 관계 |
| ■■ (중) | 2 관계 |
| ■ (약) ⚠️ | 1 관계 — 갭 후보 |
| 🔴 (없음) | 0 관계 — 확정 갭 |

### Step 3: 갭 → 브릿지 질문 생성
약한/없는 연결에서 갭을 도출하고 해결 행동을 제안

### Step 4: 우선순위 매트릭스
| 해결 비용 \ 효과 | 높음 | 중간 | 낮음 |
|-----------------|------|------|------|
| **낮음** | P0 즉시 | P1 | Skip |
| **중간** | P1 | P2 | Skip |
| **높음** | P2 | P3 | Skip |
